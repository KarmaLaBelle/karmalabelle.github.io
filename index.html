Version Control

What It Is

  • A place to store your code
  • A way to track changes in your code
  • A way to allow people to contribute to your code

Storing Your Code

Version control is typically run on a server, and acts as a sort of file transfer to allow you to dump your code into various repositories. Typically, repositories are each used to house different projects. This alone isn't what makes versioning great, however, as that would simply be a fancy FTP server, right?

Tracking Changes

Version control makes it easy to go back and see the different changes you've made over time to each and every file in your project.

Many, if not all, version control packages offer a 'Revert' function in some way or another. This enables you to go back and remove those 500 lines of code you added when someone said that adding Facebook integration was 'A Good Idea'.
